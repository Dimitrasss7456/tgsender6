
1. Общая цель
Создать софт, который:

Позволяет подключать несколько Telegram-аккаунтов по номеру телефона и коду входа (авторизация через Telegram API / Telethon / Pyrogram).

Умеет отправлять массовые рассылки в каналы, чаты и личные сообщения.

Поддерживает разные тексты и вложения для разных типов получателей (например, в каналы один текст, в чаты другой).

Имеет удобный интерфейс для управления настройками и конфигурацией рассылки.

Работает одновременно со всех подключённых аккаунтов.

2. Основные функции
2.1. Управление аккаунтами
Авторизация по номеру телефона и коду подтверждения (через Telegram API).

Возможность добавления неограниченного количества аккаунтов.

Отображение статуса каждого аккаунта (онлайн, заблокирован, ошибка).

Возможность приостановить или удалить аккаунт.

2.2. Настройки рассылки
Возможность задать разный текст для:

Каналов

Групп/чатов

Личных сообщений

Возможность прикреплять файлы (фото, видео, документы).

Планировщик отправки (задержка между сообщениями, время старта).

Лимит на количество сообщений с одного аккаунта в сутки/час.

Ротация аккаунтов (отправка очередных сообщений с разных аккаунтов для обхода лимитов).

2.3. Гибкая конфигурация
Отдельные списки для:

Каналов

Групп

Личных чатов

Возможность загрузки списков из .txt / .csv.

Настройка формата сообщения (подстановка переменных, например, {имя}, {дата}).

Автоматическая фильтрация дублей.

2.4. Интерфейс
Панель управления (настольное приложение или веб-интерфейс).

Таблицы с подключёнными аккаунтами, списками рассылок и статусом отправки.

Лог действий (успешные отправки, ошибки, баны).

2.5. Технические особенности
Использование Telegram API (Telethon или Pyrogram).

Поддержка прокси (SOCKS5, HTTP) для каждого аккаунта.

Асинхронная отправка сообщений для высокой скорости.

Возможность экспорта логов рассылки.

3. Безопасность
Данные аккаунтов (сессии Telegram) должны храниться в зашифрованном виде.

Возможность резервного копирования настроек и аккаунтов.

Поддержка работы на разных устройствах (по желанию — облачная синхронизация).

4. Дополнительно
Возможность использовать шаблоны сообщений.

Статистика: количество отправленных сообщений, доставлено/не доставлено.

Режим тестовой рассылки (отправка только на свой аккаунт для проверки).

API или Webhook для удалённого управления.

5. Требования к производительности
Поддержка от 50 до 500 аккаунтов без заметной потери скорости.

Отправка сообщений с задержкой 1–5 секунд (настраиваемой) между отправками.

Возможность параллельной работы в нескольких потоках.

6. Возможные технологии
Язык: Python (Telethon / Pyrogram) или Node.js (gramJS).

Интерфейс: PyQt5, Electron, или веб-панель (FastAPI + Vue/React).

База данных: SQLite или PostgreSQL.
